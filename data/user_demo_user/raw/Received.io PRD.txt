Received.io Application
Project Description


Recieved.io is a package and visitor tracking application for Messenger Centers.


Skyscrapers are large are beautiful buildings that house some of the biggest companies in the world.  So it shouldn’t be a surprise that security is a major concern for the property managers that operate these buildings.  The managers implemented Messenger Centers to control the flow deliveries into the building.   Every courier that delivers a package must go through the messenger center.  The messenger center manager receives the package and the package and courier information.  After a package is received by the messenger center it’s delivered to the recipient by an employee that works for the messenger center.


The purpose of received.io is to provide a digital log of packages and visitors that enter the building.


Interactions
Site Managers can do the following:


1. Enter Packages
2. Manage Tenants
3. Manage Couriers
4. Manage Floor Reps
5. Run Reports
6. Deliver a package
7. Search for packages
Floor Reps can do the following:


1. Enter Packages
2. Manage Tenants
3. Manage Couriers
4. Manage Floor Reps
5. Run Reports
6. Deliver a package
7. Search for packages


Roles


Courier -  Outside delivery person that is delivering the package to the building. 


Site Manager - Is the manager of the message center


Floor Rep -  Is the internal messenger that will deliver the package to the tenant. 


Tenant -  Is a business in the building 


Tenant Contact - An employee of the tenant. 


Package - Is the item be delivered. 


Building -  A location with many tenants.  Each building has only one site manager.   A building can have multiple floor reps. 








Type of App
Native iOS app with custom Admin Panel


App Monetization Strategy


Sass Product.   Billing will done manually by accounting department.

Functional Spec


Glossary




1. Products - The Product resource lets you update and create products.  Clients can browse products and download product specifications.


2. Classes - Valrhona host in-person tutorials throughout the year.  The Class resource lets you update and create classes.   Clients can RSVP for classes.


3. Events - Valrhona host gatherings throughout the year.   The Event resource lets you update and create events.   Clients can RSVP for events.


4. Pages - A page is an external URL that opens within an in-app browser window. 


5. Distributors - A distributor is a 3rd party company that buys products from Valrhona at a bulk discount.  They sell the product at a profit to chefs in a specific region.  Clients can search for the closest distributor. 










User Types


1. Guest - people that have no yet registered
2. Client - people that have registered and can RSVP for events and classes, download specs, and save to favorites.
3. Cercle V - inherits the capabilities of a client. Can access exclusive, products, classes, and events 
4. Admin -  Can access all screens in the Admin Panel
5. Sales Team Member -  Has read-only access to the following screens in the admin panel.
   1. Events
   2. Classes
