Project Scope
The idea of this project is to create a user-friendly web platform that will provide a secure environment to all of their registered users to view, edit/update or maintain their patient’s record within a single dashboard.

System admin will provide the restricted access control to all of their members based on their designation rights. 

Key Specifications
1. Login Page/Staff Login
Users will be able to login into the system based on the credentials provided by the system admin. Here are the details available on the login page for the users:
* Email Address/Username
* Password
* Keep me signedIn
* Forgot Password
* Submit (Button)
* Cancel (Button)


   2. Forgot Password
Users will be able to update their password by clicking on the forgot password option present on the login page. After clicking on the option, users will be redirected to a new page where he/she can enter their registered email address to receive the new link for password change. The new link contains the following options:
   * New Password
   * Confirm New Password
   * Reset (Button)

         6. Patient Records
6.1 Insurance verification - Clicking on the Insurance verification link will redirect the user to the potential new patient list that contains the relevant information of the patient displayed in a listview structure. 
If a user wants to review the detailed information about the patient then he/she can simply choose a record from the list that will redirect the user to their detailed page.

6.2 FDS Daily Entry - Clicking on the FDS daily entry menu will redirect user to the tab view structure that contains 3 options: 
        6.2.1 Patient Search - Through this tab, users can simply search the record of any patient as per their need. All the records are displayed in a list view structure.
            6.2.2 Daily Entry - Through this tab, users can simply add the claim record of any patient as per their need. All the records are displayed in a table view  structure.
           6.2.3 Checked In Patient List - Through this tab, users can simply add the claim record of any patient as per their need. All the records are displayed in a table view  structure.
6.3 Patient Data Entry/Edit: Through this tab, users can simply add the new patient record along with their claim status within the system via new patient entry form. All the records are displayed in a table view  structure.
         7. Insurance: 
             7.1 Ins. Co. Search & Edit:  Through this tab, users can search the Insurance history of any patient available within the system. All the records are displayed in a list  view  structure.
7.2 New Ins. Co. Entry:  Through this tab, users can add the new record of insurance information of any patient via Insurance Form. All the data will be saved in the master database for further process. 
There is an FAQ option under the tab menu within which users can add/edit and delete their content as per their need.
         8. Performance: 
             8.1 Event List and New Entry: Through this tab, users can search any user’s performance within a single place. All the records are displayed in a list  view  structure.
The assigned manager will have the rights to create the new report of any of their employees via “Add New Report” Button.
             8.2 Performance Summary:  Through this tab, users can search any user’s performance within a particular period of time. All the records are displayed in a list  view  structure along with the bar graph view as per user’s performance.
         9. Supply order and status: 
9.1 Supply order form: Through this tab, user can redirect in two different positions as below: 
       9.1.1  Patient: Through this tab, users can search the patient information and place the order on their behalf  via order form available. 
       9.1.2  Office: Through this tab, users can search the branch/employee information and place the order for inventory via the order form available. 
9.2 Basic Inventory: Through this tab, users can view the basic inventory list of office, medical and herbal supply information and place the order for inventory via the order form available. 
9.3 Summary Mgmt.: Through this tab, users can view the report summary of Supply order and management of Employee, Clinic manager and order manager.


10 Profile management: 
Through this tab, users can view and update their profile information within the system via Add Profile Form available for all the users based on their designation.
