change your commit messages in Git


Command
	Description
	git commit --amend


	This will open your $EDITOR and let you change the message. Continue with your usual git push origin master
	git commit --amend
git push origin master --force


	Already pushed + most recent commit
We edit the message like just above. But need to --force the push to update the remote history.






	git rebase -i HEAD~X
# X is the number of commits to go back
# Move to the line of your commit, change pick into edit,
# then change your commit message:
git commit --amend
# Finish the rebase with:
git rebase --continue


	Not pushed + old commit:
Rebase opened your history and let you pick what to change. With edit you tell you want to change the message. Git moves you to a new branch to let you --amend the message. git rebase --continue puts you back in your previous branch with the message changed
	Same as above 
git push origin master --force


	But! Remember re-pushing your commit after changing it will very likely prevent others to sync with the repo, if they already pulled a copy. You should first check with them.

Q1.  What is difference between git fetch and git pull ?


Ans-  git pull = git fetch + git merge.


1. In the simplest terms, git pull does a git fetch followed by a git merge.
2. You can do a git fetch at any time to update your remote-tracking branches under    refs/remotes    refs/remotes/<remote>/.
3. This operation never changes any of your own local branches under refs/heads, and is safe to do without changing your working copy.


Q2. Upstream vs Downstream?
        
1. Downstream - When you copy (clone, checkout, etc) from a repository. Information flowed "downstream" to you
2. Upstream - When you make changes, you usually want to send them back "upstream" so they make it into that repository so that everyone pulling from the same source is working with all the same changes.


Common error: No upstream branch found for
Solution - git branch --set-upstream master origin/master


This adds 2 parameters in .git/config :
[branch "master"] 
remote = origin 
merge = refs/heads/master
